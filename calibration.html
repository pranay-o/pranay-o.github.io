BMS Current Sensor Calibration

UBC Formula Electric · 2026

The pack measures tractive-system current through two sensors — a 400 A channel for the full range and a 50 A channel for fine resolution. Each one puts out an analog voltage, and the firmware needs to know exactly how that voltage maps to amps. This is the short Python tool that finds that mapping.

What Chimera does

The script doesn’t run on the board — it runs on a laptop and talks to the BMS over USB through Chimera, UBC Formula Electric’s Python debug tool. Chimera reaches into a running board and lets you read or drive its peripherals (ADC channels, I²C devices, GPIO) straight from Python, so you can work with real hardware without writing and flashing throwaway test firmware. Here it only needs one call — bms.adc_read(...) — to grab the live sensor voltage.

How the calibration works

Calibration is just a handful of points and a line. You run a known current through the sensor, read it off a reference meter, and type it in; the script reads both ADC channels over Chimera at the same moment. After a few points across the range, NumPy fits a straight line through them (a degree-1 Polynomial) in both directions — amps from ADC voltage, and ADC voltage from amps. The raw points go to a CSV, and it prints the slope and intercept the firmware uses.
