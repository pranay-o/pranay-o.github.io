WildCan — Wildfire Detection Web App

NwHacks 2025

WildCan is a wildfire tracker we built in a weekend at nwHacks 2025. There were four of us, all first-year students at UBC, and it was our first real hackathon. The app pulls live satellite fire data, drops every active fire onto a map, and texts you if one shows up near a location you’ve saved.

We’re from BC, where wildfire season comes around every year, so we wanted to build something people might actually use — take the raw satellite data and turn it into a text that reaches someone in time to do something about it.

How it works

Everything hangs off one MongoDB database that stores both the live fire detections and the people who’ve signed up for alerts.

  • A Python job runs on a schedule, pulls fire detections for Canada from NASA FIRMS (VIIRS satellite data), parses the CSV, and writes the latest fires into MongoDB Atlas.
  • The front end loads the map with Mapbox GL JS, centers it on BC, and pulls the current fire coordinates from the backend. Each detection gets a marker, and active fires get an animated pulsing dot on top.
  • The Get Alerts page takes a phone number and grabs your location through the browser’s geolocation API, then posts both to a Flask endpoint that checks and saves them.
  • A separate script measures the distance from each user to every active fire (using the Haversine formula), and if a fire is close enough it sends a text through Twilio.
  • There’s also a Donate button that links out to a UBC wildfire relief fund.

Get Alerts & About

Signing up is one short form: type in a phone number, hit Get Location, and submit. Those coordinates get saved and checked against each new batch of fires.

Stack

  • Front end: HTML / CSS / JavaScript, Mapbox GL JS, browser Geolocation API
  • Back end: Python (Flask) REST API with CORS, Node.js tooling
  • Data: MongoDB Atlas, NASA FIRMS satellite fire feed (VIIRS NRT)
  • Alerts: Twilio SMS, Haversine proximity check
  • Built: nwHacks 2025, team of four

What we built

We kept the scope small enough to actually finish and show off in a weekend: real satellite data going in, a live map coming out, and a real text landing on a phone. Most of the time went into wiring those pieces together — getting the FIRMS feed into MongoDB, then back out to both the map and the Twilio script — plus dealing with the browser location prompt and keeping the map smooth with a few hundred fires on it.

One honest caveat: for the demo we cranked the alert radius way up so a single test fire would reliably set off a text. In reality you’d want it closer to 50–100 km. The screenshots here are pulling live NASA FIRMS data, so the map is showing fires actually burning in BC right now.

If we kept going, the things I’d want next are sizing each fire’s radius from how strong the detection is, merging clusters of nearby detections into one fire, and real accounts so someone can save more than one location.
